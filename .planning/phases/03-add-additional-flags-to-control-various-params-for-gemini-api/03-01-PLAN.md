<objective>
Add `--temperature` and `--thinking-budget` CLI flags that control Gemini API generation parameters in both one-shot and REPL modes.

Purpose: Users currently have no way to tune how the model reasons or how creative its answers are. This phase exposes the two most impactful generation parameters as long-form CLI flags, consistent with the existing `--show-refs` style.

Output:
- `internal/gemini/client.go` — `NewClient()` accepts a `ClientConfig` struct; sets `Temperature` and optionally `ThinkingConfig` on `GenerateContentConfig`
- `cmd/ais/main.go` — two new flags, preset validation/mapping for thinking budget, temperature range check, threading to both call sites
- `internal/repl/repl.go` — `Run()` signature updated to accept `ClientConfig`, passed to `NewClient()`
</objective>
