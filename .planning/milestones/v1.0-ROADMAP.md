# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-23
**Phases:** 1–3
**Total Plans:** 7

## Overview

Build a Go CLI tool that wraps the Gemini API with always-on Google Search grounding. Phase 1 delivers a working tool — both query modes running end-to-end with rendered output and source citations. Phase 2 hardens it — error handling, lint compliance, and correct build targets. Phase 3 adds user-tunable generation parameters.

## Phases

### Phase 1: Working Tool

**Goal**: A developer can query Gemini with Google Search grounding from the terminal in both one-shot and chat modes
**Depends on**: Nothing (first phase)
**Requirements**: MODE-01, MODE-02, MODE-03, SRCH-01, SRCH-02, OUT-01, OUT-02, CFG-01, TOOL-01
**Plans**: 4 plans

Plans:
- [x] 01-01: Foundation — dependencies, Gemini client with grounding, render helpers, Makefile fix
- [x] 01-02: One-shot mode — `-q` flag, spinner, glamour render, source citations
- [x] 01-03: Chat REPL — interactive loop, multi-turn history, `ais> ` prompt
- [x] 01-04: End-to-end verification — build checks + live API human checkpoint

**Success Criteria (all TRUE at ship):**
1. `ais -q "query"` prints a markdown-rendered answer with source URLs, then exits cleanly
2. `ais` (no args) opens an interactive REPL and accepts successive queries without restarting
3. In chat mode, follow-up questions reference earlier answers (multi-turn context preserved)
4. Responses are visually rendered — headers, bold, code blocks — not raw markdown strings
5. Source URLs from Google Search grounding appear after each response body
6. `make build` produces `./bin/ais`

**Key decisions:**
- New `google.golang.org/genai` SDK used (not deprecated `github.com/google/generative-ai-go`)
- `glamour.WithAutoStyle()` for automatic dark/light terminal theme detection
- Single `gemini.Client` reused across REPL turns to preserve `ChatSession` history

### Phase 2: Production Ready

**Goal**: The tool handles failure gracefully and meets production code quality standards
**Depends on**: Phase 1
**Requirements**: CFG-02, ERR-01, ERR-02, ERR-03, TOOL-02
**Plans**: 2 plans

Plans:
- [x] 02-01: Error handling — missing key message, API error classification, empty query guard
- [x] 02-02: Lint compliance — replace `fmt.Print*` with `fmt.Fprint*`, make lint zero violations

**Success Criteria (all TRUE at ship):**
1. `ais` without `GEMINI_API_KEY` shows actionable message — no panic, no cryptic output
2. Network failure or API error displays failure reason with suggested next step
3. Unknown flags or empty input show usage help, not a stack trace
4. `make lint` passes with zero golangci-lint violations

**Key decisions:**
- Duplicate `classifyAPIError` in both `main.go` and `repl.go` (avoided changing `Run()` signature)
- Substring matching on SDK error message strings (SDK doesn't expose typed sentinel errors)

### Phase 3: Add Additional Flags to Control Gemini API Params

**Goal**: Users can tune temperature and thinking budget at invocation time via `--temperature` and `--thinking-budget` flags, applied to both one-shot and REPL modes
**Depends on**: Phase 2
**Requirements**: CFG-03, CFG-04
**Plans**: 1 plan

Plans:
- [x] 03-01: Add `--temperature` and `--thinking-budget` flags — `gemini.ClientConfig`, validation, threading through `main.go` and `repl.go`

**Success Criteria (all TRUE at ship):**
1. `--temperature` accepts float [0.0, 2.0], defaults 0.7, applied via `GenerateContentConfig`
2. `--thinking-budget` accepts presets (none/low/medium/high/auto), defaults auto
3. Both flags apply in one-shot and REPL modes
4. Out-of-range temperature exits 1 with clear error; invalid preset exits 1 with valid values list

**Key decisions:**
- Pre-validated config struct pattern: caller validates, gemini package consumes without re-checking
- `ThinkingBudget` sentinel `-1` = auto (omit `ThinkingConfig`); any other value sets explicit budget
- CLI preset → token mapping via `map[string]int32`

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- Single SDK (`google.golang.org/genai`) throughout — no abstraction layer needed at this scale
- Always-on Google Search grounding — no per-query toggle simplifies UX
- Duplicate `classifyAPIError` across packages over interface change
- Pre-validated `ClientConfig` struct as boundary between CLI flags and API params

**Issues Resolved:**
- Gemini SDK migration (generative-ai-go → genai) resolved at Phase 1 start
- `fmt.Print*` forbidigo lint violations resolved in Phase 2

**Issues Deferred:**
- Streaming output (OUT-03) — deferred to v2
- Stdin pipe support (INP-01) — deferred to v2
- System prompt flag (CUST-01) — deferred to v2

**Technical Debt Incurred:**
- Duplicate `classifyAPIError` helper in `main.go` and `repl.go` — acceptable for now, refactor if a third call site emerges
