# Requirements Archive: ais v1.0

**Archived:** 2026-03-23
**Milestone:** v1.0 MVP
**Core Value:** Every query returns a grounded, source-cited answer — either as a quick one-liner or a full conversation — without leaving the terminal.

## v1 Requirements — All Complete ✅

### Query Modes

- [x] **MODE-01**: Running `ais -q "query"` returns an answer and exits (one-shot mode) — *Validated Phase 1*
- [x] **MODE-02**: Running `ais` with no args opens an interactive REPL (chat mode) — *Validated Phase 1*
- [x] **MODE-03**: In chat mode, the full conversation history is passed to Gemini on each turn (multi-turn context) — *Validated Phase 1*

### Search Grounding

- [x] **SRCH-01**: Google Search grounding is enabled on every query (no toggle needed) — *Validated Phase 1*
- [x] **SRCH-02**: Grounding sources/URLs are listed after each response — *Validated Phase 1*

### Output

- [x] **OUT-01**: Responses are rendered as markdown in the terminal (via glamour) — *Validated Phase 1*
- [x] **OUT-02**: Source citations appear after the rendered response body — *Validated Phase 1*

### Configuration

- [x] **CFG-01**: GEMINI_API_KEY environment variable is used for authentication — *Validated Phase 1*
- [x] **CFG-02**: Missing API key produces a clear, actionable error message — *Validated Phase 2*
- [x] **CFG-03**: `--temperature` flag controls sampling temperature in [0.0, 2.0], default 0.7, applies to both one-shot and REPL modes — *Validated Phase 3*
- [x] **CFG-04**: `--thinking-budget` flag controls reasoning token budget via presets (none/low/medium/high/auto), default auto, applies to both modes — *Validated Phase 3*

### Error Handling

- [x] **ERR-01**: Missing API key error tells user exactly what to set — *Validated Phase 2*
- [x] **ERR-02**: Network/API errors show the failure reason and suggest next steps — *Validated Phase 2*
- [x] **ERR-03**: Unknown flags or bad input show usage help — *Validated Phase 2*

### Tooling

- [x] **TOOL-01**: `make build` produces `./bin/ais` binary — *Validated Phase 1*
- [x] **TOOL-02**: Code passes existing golangci-lint rules — *Validated Phase 2*

| **CUST-01**: `-s` system prompt flag | Nice-to-have — v2 feature |

## Out of Scope (Remained)

| Feature | Reason |
|---------|--------|
| Config file | Env var is sufficient; config file adds complexity without value |
| Multiple AI providers | Gemini-only keeps the scope tight |
| Persistent conversation history | Sessions are ephemeral; disk storage is v2+ |
| Web UI / TUI | Terminal text output is the right interface |
| OAuth / complex auth | API key is the right auth model for a personal CLI tool |
